ARCHITECTURE SUMMARY (for NotebookLM context)
════════════════════════════════════════════════════════════════

FRONTEND (React 19 / Vite / TypeScript)
  Strict "Control Panel" — no direct exchange connections,
  no market data fetch loops, no trading execution logic.
  Communicates exclusively through Supabase.

SUPABASE (Data & Serverless Layer)
  strategy_registry      ← frontend writes bot configs
  system_status          ← frontend writes kill-switch flag
  trade_logs             ← worker writes; frontend reads realtime
  performance_evaluator  ← worker writes; frontend reads realtime
  45 Edge Functions      ← Deno workers for secure server-side ops

CORE BRAIN (Python / Render Background Worker)
  core-brain/trading_worker.py
  Continuous while-True loop (default 60s):
    1. Read system_status.active — halt immediately if false
    2. Read strategy_registry WHERE is_active = true
    3. Fetch live prices from CoinGecko (one call per cycle)
    4. Per-strategy frequency gate (realtime/minute/hourly/daily)
    5. Simulate trade → insert into trade_logs
    6. Recalculate metrics → upsert performance_evaluator

KEY DEPENDENCIES
  Frontend:  react@19, @supabase/supabase-js@2, recharts@3,
             @tanstack/react-query@5, tailwindcss@3, radix-ui
  Worker:    supabase-py, requests, pandas, python-dotenv
