## Top priorities (in order)

1. **Rotate `SUPABASE_ACCESS_TOKEN`** (compromised by deleted `get-secrets.yml` workflow) and add to GitHub Actions secrets so edge-function deploys actually work.

2. **Rotate Supabase JWT secret** (if any chance the leak workflow ran). Update consumers:
   - Vercel env vars (`VITE_SUPABASE_PUBLISHABLE_KEY`)
   - Render env vars (`SUPABASE_SERVICE_ROLE_KEY`)
   - 6 workflow files that hardcode anon key (centralize into GitHub Actions secret in same pass)

3. **`STRIPE_SECRET_KEY` + `STRIPE_WEBHOOK_SECRET` into Render + Supabase**. Per CLAUDE.md, secret key was recovered to a local `.env.secrets` file but not yet pushed to platforms. Without these, checkout + webhook flow are broken.

4. **Decide live-trading posture**: with `ALPACA_PAPER_MODE=false`, current state is live but no Alpaca keys = endpoints will fail. Either flip to `true` and run safely in paper mode, or add real keys WITH the safety review in the audit before flipping.

5. **Broker keys (TRADIER/BINANCE/KRAKEN/IBKR)**: only add if you actually intend those brokers in the initial launch. Otherwise leave unset — endpoints fail with helpful 503, which is correct behavior.

6. **Wallet private keys (HYPERLIQUID/SOLANA)**: only add when momentum-bot, flash-bot, yield-scanner are intended to run live. Until then, those workers will start, fail to find keys, and crash-loop on Render — which is fine (no funds at risk).

7. **NEVER**: paste any of these into a chat, commit them to git, post in Slack/Discord, or share with anyone who doesn't operationally need them.
